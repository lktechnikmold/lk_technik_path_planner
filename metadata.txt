about=Plugin for importing and exporting guidance and field data for agricultural steering systems.
    The export format is chosen automatically from the selected terminal (ISOXML v3/v4, John Deere Gen4, AgGPS).
    Features a field catalogue (Felder.csv) as the single source of truth, a structured selection tree (Customer → Farm → Field), field management (add, rename, delete, dropdown selection), optional contour segments (Fendt One), and AgGPS import/export.

changelog=2.0.0 - Veroeffentlichung: Feld-Katalog, Terminal-Auswahl, AgGPS
    - Feld-Katalog (Felder.csv) als zentrale Quelle: Felder ohne Grenze und mehrere Grenzen pro Feld; Felder anlegen/umbenennen/loeschen; Feld-Auswahl per Dropdown.
    - Terminal-Auswahl bestimmt das Exportformat (ISOXML v3/v4, John Deere Gen4, AgGPS); Kontursegmente nur bei Fendt One.
    - AgGPS-Export und -Import (Ordnerstruktur mit boundary/swaths/AreaFeature/PointFeature.shp).
    - Diverse Stabilitaets-Fixes; Feldflaeche wird immer aus der Geometrie berechnet.
    1.10.1 - AgGPS-Import Fix
    - Feldflaeche wird beim Import immer aus der Geometrie berechnet (nie aus der Datei).
    1.10.0 - AgGPS-Import
    - Import der AgGPS-Ordnerstruktur (Data/Kunde/Betrieb/Feld) mit boundary/swaths/AreaFeature/PointFeature.shp.
    - Wird automatisch erkannt, wenn ein AgGPS-Ordner gewaehlt wird.
    1.9.1 - AgGPS-Export Fixes
    - Fix: MultiLineString-Fahrspuren verursachten einen Fehler (Geraden/Kurven-Erkennung jetzt robust ueber Stuetzpunkt-Anzahl).
    - boundary.shp enthaelt zusaetzlich die Spalte Area (Flaeche in m2).
    1.9.0 - AgGPS-Export
    - Export ins AgGPS-Format (Trimble/Case/New Holland u.a.): AgGPS/Data/Kunde/Betrieb/Feld/ mit boundary.shp, swaths.shp, AreaFeature.shp, PointFeature.shp.
    - swaths.shp: Spalte id (0 = Gerade, 1 = Kurve), Spalte Name.
    1.8.0 - Terminal-Auswahl bestimmt das Exportformat
    - Statt v3/v4/Gen4/Kontursegmente jetzt ein Terminal-Dropdown (Marke + Modell); das Dateiformat wird automatisch gesetzt.
    - Kontursegmente-Option erscheint nur bei Fendt One.
    - AgGPS-Terminals (Trimble/Case/New Holland) werden erkannt, der Export ist aber noch nicht implementiert (Hinweis).
    1.7.0 - Namensmodell & Stabilisierung
    - Feldname = Name der ersten Feldgrenze (oder via "Feld hinzufügen"); weitere Grenzen behalten eigene Namen.
    - Export: PFD traegt den Feldnamen, jede Feldgrenze eine eigene PLN mit eigenem Namen.
    - Fix: kein OGR-Fehler / keine Doppelanlage mehr bei offenem Bearbeitungsmodus.
    - Fix: Feld-Dropdown erscheint sofort nach Import und in Altprojekten.
    1.6.0 - Felder loeschen
    - Rechtsklick "Feld loeschen..." mit Sicherheitsabfrage; entfernt Grenzen, Fahrspuren, Hindernisse und Katalogeintrag.
    1.5.0 - Feld-Auswahl per Dropdown
    - ID-Feld von Feldgrenzen/Fahrspuren/Hindernissen als Value-Relation-Dropdown (zeigt Feldnamen, speichert id).
    1.4.0 - Felder anlegen, abgleichen, umbenennen
    - Button "Feld hinzufuegen" fuer Felder ohne Grenze.
    - Katalog-Abgleich beim Oeffnen (inkl. IDs aus Fahrspuren/Hindernissen, Auto-ID fuer neue Grenzen).
    - Rechtsklick "Feld umbenennen...".
    1.3.0 - Feld-Katalog & robuster Export
    - Neuer Feld-Katalog Felder.csv (id;Name) pro Betrieb als Source of Truth.
    - Import (ISOXML & John Deere) schreibt alle Felder in den Katalog, auch ohne Feldgrenze.
    - Export laeuft ueber den Katalog: Felder ohne Grenze werden exportiert, mehrere Grenzen pro Feld moeglich.
    - Auswahlbaum zeigt Katalog-Felder; Unique-Constraint auf ID von hart auf weich (nur Warnung).
